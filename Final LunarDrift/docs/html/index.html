Lunar Drift Documentation

Hello, developer! Welcome to the system documentation of Lunar Drift.

Lunar Drift's engine employs a variety of different approaches working together.

  • Component-Based Architecture: Components can be plugged into SceneObjects to control the behaviour in an abstract manner.
    View components here.
  • Inheritance-Based Architecture: Object-orientedness is used as a primary paradigm for encapsulation.
    View graphical view of inheritance hierarchy here.
  • Publish-Subscribe Pattern: Messages are used to reduce cyclic dependencies, and modularize inter-component communication.
    View messages here.
  • Managers: Managers are used to encapsulate behaviours, and/or handle resource management effectively.
    View managers here.

Otherwise, to begin, on the top menu you can go to Classes to view the classes in the project alphabetically for your viewing; a graphical Class Hierarchy is also an option as visual overview of the project's classes.

You can also view the top-level list of Modules for an organised break-down of the engine structure, which is likely easier to navigate than the list of classes.

If you're only here for managing the front-end game, the best place to go would be the list of front-end game classes. Otherwise, if you're managing the code in the back of the engine, be sure to acquaint yourself with everything else, if not the modules of classes that are appropriate.

As a final alternative, you can go to Files to view the project structure of Lunar Drift, where extended documentation of each file can be displayed.

For additional queries about the project, contact the lead programmer and project manager, Hayley Hatton.
